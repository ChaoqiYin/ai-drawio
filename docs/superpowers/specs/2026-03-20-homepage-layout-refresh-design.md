# 首页布局优化设计

## 背景

当前首页已经拆成左右两栏，但左侧仍承担了过多操作职责，导致：

- 左侧标题文案偏长，在当前宽度下容易占两行。
- “创建本地会话”“清空全部本地数据”作为高频操作，放在左侧会打断右侧列表区的主流程。
- 左侧面板在移除会话数量标签后出现明显留白，视觉重心不足。

本次改动的目标是让首页的信息层级更清楚：左侧负责引导和视觉氛围，右侧负责搜索、操作和列表。

## 目标

- 将“新建会话”“清空全部本地数据”移动到右侧控制区。
- 将左侧标题压缩为稳定单行，并使用更精简的产品化文案。
- 为左侧补充装饰性 SVG，填补留白并增强画布/结构图语义。
- 保持当前会话列表、搜索、分页、重命名、删除、跳转遮罩和设置按钮的既有行为。

## 非目标

- 不调整会话数据模型、分页逻辑或跳转逻辑。
- 不改动设置入口的位置和交互。
- 不引入新的插画资源文件或图片依赖，SVG 直接以内联方式实现。

## 选定方案

采用“视觉左栏 + 右侧控制行”的双栏方案。

原因：

- 最符合“左侧弱操作、右侧强操作”的任务流。
- 能稳定满足标题单行、按钮右移、左侧补足视觉留白这三个核心要求。
- 对现有状态管理和列表交互侵入最小，适合在当前页面结构上迭代。

## 页面结构

### 左侧面板

左侧保留固定宽度面板，但职责调整为“引导面板”。

- 顶部增加很轻的 eyebrow 文本，用于补充语义层级。
- 主标题改为：`继续你的绘图工作`
- 副文案改为：`选择一个会话，回到上次的画布。`
- 标题和副文案放在同一信息块中，标题使用单行截断或稳定宽度控制，避免折成两行。
- 面板下半区放一个装饰性 SVG，不承载任何交互。

### 右侧面板

右侧继续作为主工作区，但顶部控制区升级成一条统一工具行。

- 左侧放搜索框。
- 右侧依次放“新建会话”“清空全部本地数据”。
- 分页信息保留在控制区下方，仍然服务会话列表。
- 会话列表和卡片交互保持现有行为。

## 文案

本次首页文案统一往更短、更直接的方向收敛。

- 左侧标题：`继续你的绘图工作`
- 左侧说明：`选择一个会话，回到上次的画布。`
- 右侧主按钮文案可保留现有“创建本地会话”，也可在实现时视空间微调为“新建会话”；优先保证语义清晰和横向排布稳定。

## SVG 设计方向

SVG 采用浅蓝灰色系的抽象图形语言，呼应 draw.io 的画布、节点、卡片和连线语义。

视觉原则：

- 仅作为装饰，不出现按钮式视觉暗示。
- 保持低对比度，不能抢过右侧会话列表。
- 通过圆角矩形、圆点、连接线和柔和渐变构造“结构图”氛围。
- 在桌面宽度下优先填充左侧下半区；在较窄宽度下允许缩小，但不应挤压标题文案。

实现方式：

- 直接在 `conversation-home.tsx` 中以内联 SVG 输出。
- 不新增静态资源文件，避免额外构建和路径处理。

## 布局与响应式要求

- 维持当前桌面双栏结构，不退回单栏 hero 布局。
- 左侧面板宽度可从当前固定值做小幅优化，但仍保持稳定的视觉宽度，不随着列表内容抖动。
- 右侧控制区需要支持横向排列；当可用宽度不足时，允许按钮换行到下一行，但搜索框始终保留最高优先级。
- 列表区继续保持 `flex-1 min-h-0 overflow-y-auto` 的主滚动策略。

## 交互与状态约束

- “新建会话”继续复用现有 `handleCreateConversation`。
- “清空全部本地数据”继续复用现有 `handleClearAllData` 和 `Popconfirm`。
- 搜索禁用、删除禁用、遮罩态、重命名弹窗、空态提示等现有状态行为不变。
- 左侧 SVG 和引导文案不引入任何新的状态依赖。

## 测试与验证

本次改动以现有 source-level 回归测试为主，并补充对新布局结构的断言。

至少覆盖：

- 左侧不再渲染主操作按钮。
- 右侧控制区包含搜索框与两个操作按钮。
- 左侧标题和说明文案更新为新的精简版本。
- 左侧包含内联 SVG 或明确的视觉容器标记。
- 既有分页和列表结构断言继续成立。

## 风险与处理

### 标题仍然换行

如果左侧宽度不足或标题样式过大，标题仍可能断行。实现时应优先通过更短文案、明确宽度和合适字号解决，而不是继续缩窄面板内边距。

### 右侧控制区过挤

如果搜索框、按钮和分页同时挤在同一水平层会显得拥挤，应采用“工具行 + 分页次级行”的两层结构，而不是牺牲按钮可点面积。

### SVG 抢视觉

如果 SVG 对比过强，会削弱右侧列表的主任务感。实现中应控制颜色、透明度和元素密度，以“补足留白”为上限。

## 实施范围

主要修改文件：

- `app/(internal)/_components/conversation-home.tsx`
- `tests/conversation-home-source.test.ts`
- 如有必要，可补充新的首页布局 source test，但应优先复用现有测试文件。
